Account (1)

Download OpenAPI specification:Download

Developers Relationship Team: developers@onne.place URL: https://developers.onne.place/ License: Unlicensed

API for handling Accounts maintenance and operations of currency debits, credits and transfers.

The current focus of this API currently provides all reading operations, mainly what is needed to provide a complete statement of the current balance and transactions.

In the future this may be just an facade on top of a accounting product.

Design

Various inputs were taken to design this API, including some banking as a service APIs (Mambu), Brazilian Pix and the UK OpenBanking.

Also many ISO standards where taken into account like:

  • 4217 for Currencies
  • 3166 for Country Codes
  • 639-2 for Languages

Decisions

This service interacts directly with the database and is supposed to be the last mille of interaction with accounts. At this point no major optimization is planned to scale this up, beyond service horizontal scaling and database vertical scaling. A simple scaling measure could be partitioning instances of the service and database, per branch or set of accounts.

Rich transactions that require external interactions, like payments, should be done in a separate service and use SAGA pattern to ensure transactional consistency.

required
string <uuid> = 36 characters
Example: 32e04e84-17fd-4e34-a926-c64d92647673

Idempotency Key used to guarantee the operation to be executed no more than once. Defined at The Idempotency-Key HTTP Header Field

Request Body schema: application/json

Data of the transaction being registered

transaction_type_id
string <uuid> = 36 characters

The type of this transaction

status
string
Enum: "pending" "booked" "deleted"

Every transaction can be in one of the following states:

  • pending: Transaction was posted but not yep processed and booked.
  • booked: Transaction is effective and final.
  • deleted: Transaction witch was not booked yet, was removed.
counterpart_account_id
string <uuid> = 36 characters

The counterpart to this transaction. A unique and immutable identifier used to identify the account resource. This identifier has no meaning to the account owner.

amount
number <decimal>

Amount expressed as a decimal number of major currency units

notes
string [ 1 .. 128 ] characters .*

A text free description of the transaction.

external_transaction_id
string [ 1 .. 35 ] characters ^[a-zA-Z0-9]{1,35}$

Transaction identification, known by the user and used for transaction reconciliation across institutions.
